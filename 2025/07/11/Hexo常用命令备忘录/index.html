我的第一篇博客：Hexo、Git与NPM常用命令终极指南

欢迎来到我的新博客——《林子的写生簿》！

在敲下这些字的时候，我正坐在自己的“数字小屋”里。窗外是浩瀚的互联网，屋内是我亲手布置的一桌一椅。从决定拥有这样一个空间，到它最终落成，我经历了一段充满探索、迷茫、求助，最终豁然开朗的旅程。

这篇开篇文章，我想记录的不仅是那些让我头疼又让我成长的命令，更是这段旅程本身。如果你也曾或正打算开启这样一段旅程，希望我的这份“施工日志”能给你带来些许慰藉和勇气。

一、 Hexo 核心工作流 (The Essentials)

这一部分是博客的“日常”，是我们每天都会用到的核心命令。

hexo new：创建新内容

这是我们写作的起点。

# 创建一篇新的文章，它会出现在你的博客文章列表里
hexo new "我的第一篇文章"

# 创建一个独立的页面，比如“关于我”或“友情链接”
hexo new page "about"
hexo server：开启“实时预览”模式

在本地启动一个一模一样的“镜像网站”来预览所有改动。

Bash
# 启动服务器，可简写为 hexo s
hexo server

# 如果4000端口被占用，可以指定一个新端口
hexo server -p 5000
运行后，在浏览器访问 http://localhost:4000 (或你指定的新端口)，即可看到你的博客。

hexo generate：生成静态文件

将你的 Markdown 文章、主题文件等“原材料”，编译成浏览器真正认识的 HTML、CSS、JS 静态网页文件。

Bash
# 生成网站文件，可简写为 hexo g
hexo generate

# 在生成时监听文件变动，适合在写主题时使用
hexo generate --watch
hexo deploy：部署到线上

将生成好的 public 文件夹里的所有内容，推送到你在 _config.yml 里配置好的 GitHub 仓库。

Bash
# 部署网站，可简写为 hexo d
hexo deploy
hexo clean：清理缓存

在重新生成网站前，执行这个命令可以删除旧的 public 文件夹和数据库缓存，避免因缓存造成一些奇怪的问题。

Bash
hexo clean
黄金组合：一键发布

在实际使用中，我们最常用的是将 generate 和 deploy 组合起来。

Bash
# 先清理，再生成，最后部署，一气呵成！
hexo clean && hexo g -d
二、 Hexo 进阶与维护 (Advanced Usage)
当博客遇到一些小问题，或者你想进行更深入的管理时，这些命令会派上用场。

hexo list：列出网站内容

快速查看你的网站都包含了哪些路径、页面和文章。

Bash
# 列出所有页面
hexo list page

# 列出所有文章
hexo list post

# 列出所有分类和标签
hexo list category
hexo list tag
hexo version：查看版本信息

当你需要向别人求助，或者查看某个功能是否受支持时，这个命令可以告诉你当前环境的所有版本信息。

Bash
hexo version
hexo --debug：开启调试模式

当 hexo g 或 hexo d 报错时，用这个模式可以打印出非常详细的日志，帮助你定位问题。

Bash
hexo g --debug
三、 NPM 包管理 (Package Management)
Hexo 的很多功能（如部署器、渲染器）都是通过插件（NPM包）来实现的。

npm install：安装依赖

当我们拿到一个旧的博客项目，或者 node_modules 文件夹丢失时，这个命令会根据 package.json 文件自动下载所有需要的插件。

Bash
# 简写为 npm i
npm install
npm install <plugin>：安装新插件

为你的博客添加新功能。

Bash
# 安装 Git 部署器插件，这是 hexo d 命令能用的前提
npm install hexo-deployer-git --save

# 安装 RSS 订阅生成插件
npm install hexo-generator-feed --save
小贴士： --save 参数会自动把这个插件信息记录到你的 package.json 文件里，这是一个好习惯。

npm uninstall <plugin>：卸载插件

当你不再需要某个插件时。

Bash
npm uninstall hexo-generator-feed --save
npm update：更新所有插件

尝试将所有插件更新到最新版本。

Bash
npm update
四、 Git 与主题管理 (Version Control)
Git 不仅用于部署，也用于管理你的主题和博客源文件。

git clone：下载主题

这是我们给博客“换肤”时用的命令。

Bash
# 把一个主题从 GitHub 下载到 themes 文件夹下，并命名为 typing
git clone [https://github.com/geekplux/hexo-theme-typing.git](https://github.com/geekplux/hexo-theme-typing.git) themes/typing
git pull：更新主题

如果你的主题作者发布了新版本，你可以进入主题的文件夹，用这个命令来获取更新。

Bash
# 首先，进入你的主题文件夹
cd themes/typing

# 拉取最新的更新
git pull
五、 终端与文件系统 (The Foundation)
这些是我们解决问题时用到的“瑞士军刀”。

cd <folder>: 切换目录。走进不同的“房间”（文件夹）。

ls -l: 列出文件。查看当前“房间”里都有什么东西，以及它们的详细信息。

pwd: 显示当前位置。在你迷路时，告诉你正身处何方。

sudo chown -R $(whoami) <folder>: 更改所有权。这是我们解决权限问题的终极武器，将文件夹的“所有权”交还给你自己。

rm -rf <folder/file>: 强制删除。这是一个非常强大的命令，可以删除文件或文件夹。请务必谨慎使用，因为它没有回收站！

总结与展望
以上就是我在这次建站之旅中学到的核心命令，从日常写作到进阶维护，基本涵盖了所有常用场景。它们看似零散，但串联起来，就构成了一套优雅而高效的数字内容创作流程。

掌握它们，就等于掌握了这间数字小屋的建造和维护能力。

这篇“说明书”就到这里。现在，这本“写生簿”已经准备好了，期待未来能用文字和思考将它慢慢填满。

旅途愉快。
